## Fluxo Detalhado de uma Requisiçăo

```
1. CLIENTE ENVIA REQUISIÇĂO
   ?? POST /api/clientes
      ?? { "nome": "Joăo", "telefone": "(11)...", "endereco": "..." }

2. MINIMAL API RECEBE
   ?? ClienteEndpoints.CreateClienteEndpoint()
      ?? Recebe CreateClienteDto (DTO)

3. CRIA COMMAND
   ?? CreateClienteCommand(nome, telefone, endereco)

4. DESPACHA VIA WOLVERINE
   ?? await messageBus.InvokeAsync<ClienteResponseDto>(command)

5. WOLVERINE EXECUTA MIDDLEWARE DE VALIDAÇĂO
   ?? ValidationMiddleware<CreateClienteCommand>
      ?? CreateClienteValidator.ValidateAsync()
         ?? Valida Nome (obrigatório, 3-150 chars)
         ?? Valida Telefone (formato brasileiro)
         ?? Valida Endereço (5-255 chars)
         
         Se inválido:
         ?? Retorna 400 Bad Request com errors

6. HANDLER PROCESSA
   ?? CreateClienteCommandHandler.Handle(command)
      ?? ClienteMapper.ToEntity(command)
      ?  ?? Mapeia CreateClienteCommand ? Cliente (automático Maperly)
      ?? repository.AddAsync(cliente)
      ?  ?? Adiciona ao DbContext (ainda em memória)
      ?? repository.SaveChangesAsync()
      ?  ?? Executa INSERT no SQL Server
      ?     ?? Gera ID único (Guid)
      ?     ?? Define DataCriacao
      ?? ClienteMapper.ToResponseDto(clienteAdicionado)
         ?? Mapeia Cliente ? ClienteResponseDto

7. RESPOSTA RETORNA
   ?? 201 Created
      ?? Location: /api/clientes/{id}
      ?? Body: { "id": "...", "nome": "Joăo", ... }

8. CLIENTE RECEBE JSON
   ?? Application exibe ou armazena dados
```
